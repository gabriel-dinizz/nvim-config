## Shell Changes Explained

### zsh-syntax-highlighting

This is what changed the colors of your terminal commands. It colorizes text as you type:

- **Green** — valid/recognized commands (e.g., `ls`, `cd`, `git`)
- **Red** — unrecognized or misspelled commands
- **Underlined** — valid file paths
- **Strings in quotes** — highlighted differently

This is purely visual and does not change how commands work.

**To disable:** Remove `zsh-syntax-highlighting` from the plugins line in `~/.zshrc` and run `source ~/.zshrc`.

### zsh-autosuggestions

This is what shows grayed-out suggestions as you type, based on your command history.

| Action | Key |
|--------|-----|
| Accept full suggestion | `→` (right arrow) |
| Accept next word only | `Ctrl+→` (ctrl + right arrow) |
| Dismiss suggestion | Keep typing or press `Esc` |

The suggestions come from your shell history — the more you use the terminal, the better they get.

**To disable:** Remove `zsh-autosuggestions` from the plugins line in `~/.zshrc` and run `source ~/.zshrc`.

### web-search

Allows you to search the web directly from the terminal.

| Command | Action |
|---------|--------|
| `google kittens` | Opens browser and searches Google for "kittens" |
| `ddg kittens` | Searches DuckDuckGo |
| `bing kittens` | Searches Bing |

### PowerLevel10k

A theme for Oh My Zsh that customizes your terminal prompt. Was already configured in `~/.zshrc` (`ZSH_THEME="powerlevel10k/powerlevel10k"`). Run `p10k configure` to reconfigure its appearance.

## Format-on-Save Configuration

Formatting runs automatically when you save a file (`:w`):

| Language | Formatter | Install |
|----------|-----------|---------|
| Python | black | `pip install black` |
| Lua | stylua | `brew install stylua` |
| C/C++ | clang-format | Included with clangd |

If the formatter is not installed, it falls back to LSP formatting.

## First-Time Setup

1. Open a new terminal to activate the zsh plugins, or run `source ~/.zshrc`
2. Run `p10k configure` to set up the PowerLevel10k terminal theme
3. Open neovim: `nvim`
4. Run `:PackerSync` to ensure all plugins are up to date
5. Run `:MasonUpdate` to update the Mason registry
6. Treesitter parsers auto-install when you open supported file types

## How to Undo These Changes

### Remove zsh plugins

Edit `~/.zshrc` line 80 and change back to:
```bash
plugins=(git)
```
Then run `source ~/.zshrc`. Optionally delete the plugin directories:
```bash
rm -rf ~/.oh-my-zsh/custom/plugins/zsh-autosuggestions
rm -rf ~/.oh-my-zsh/custom/plugins/zsh-syntax-highlighting
```

### Remove Neovim

```bash
brew uninstall neovim
rm -rf ~/.config/nvim
rm -rf ~/.local/share/nvim
rm -rf ~/.local/state/nvim
rm -rf ~/.cache/nvim
```

### Remove ripgrep

```bash
brew uninstall ripgrep
```

## Compatibility Fixes Applied (Neovim 0.11)

Neovim 0.11 introduced breaking changes that affect several plugins. The following fixes are applied in `init.lua`:

### lspconfig deprecation warning suppressed

Neovim 0.11 introduced a new built-in `vim.lsp.config` API, deprecating the old `require('lspconfig')` approach. The `nvim-lspconfig` plugin hasn't migrated yet, so `vim.deprecate` is patched to suppress the warning. The plugin still works correctly.

### Telescope `ft_to_lang` / `is_enabled` fix

The Telescope `0.1.x` branch called `nvim-treesitter.parsers.ft_to_lang()` and `is_enabled()`, both of which were removed in newer versions of `nvim-treesitter` and Neovim 0.11. This caused errors when previewing files with `<Space>ff`.

**Fix:** Telescope was updated from the `0.1.x` branch to `master` (latest), which uses the current APIs. A compatibility shim for `ft_to_lang` is also included in `init.lua` as a safety net for any other plugins that may reference it.

### `vim.tbl_islist` warning

This is a cosmetic deprecation warning from Packer. It does not affect functionality and will be resolved in future plugin updates.

- Treesitter parsers compile on first use, which may cause a brief delay when opening a new file type for the first time.
